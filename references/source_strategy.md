# 数据源与核验策略

本文件用于旅行攻略的信息采集阶段。原则是：先找稳定来源，再补充社交平台热度；动态数据必须标注查询时间和可信度。

## 推荐来源优先级

### 天气

- 优先：中央气象台、中国天气、地方气象局。
- 补充：天气网、墨迹等商业天气服务。
- 输出：日期、天气、温度、降雨概率、风力、穿衣和雨天调整建议。

### 景点与开放信息

- 优先：景区官网、官方公众号/小程序、文旅局页面。
- 补充：携程、去哪儿、飞猪、马蜂窝、穷游。
- 输出：开放时间、门票、预约规则、游玩时长、最佳时段、雨天影响。
- 若只能从攻略站或搜索摘要获取，标注为「搜索线索」。

### 社交热度与避雷

- 点位发现、拍照机位、近期热度和避雷反馈优先使用小红书。
- 补充来源：抖音、快手、微博、B站、马蜂窝游记。
- 用途：发现近期热门打卡点、拍照机位、避坑反馈、本地玩法。
- 注意：社交平台内容不可作为价格和开放时间的唯一依据。
- 点位推荐阶段默认不抓取每一篇小红书帖子。输出点位时附小红书搜索结果页入口即可，让用户自己看图和笔记。
- 小红书搜索结果页优先使用 `https://www.xiaohongshu.com/search_result_ai?keyword={URL编码后的关键词}&source=web_explore_feed`；如果该入口不可用，降级为 `http://www.xiaohongshu.com/search_result/?keyword={URL编码后的关键词}`。
- 只有用户明确要求深挖某个点位时，才尝试抓取网页端可见帖子内容，例如标题、作者、发布时间、正文摘要、互动热度和可见评论摘要。不能保证抓到 App 专属内容、私密内容、被风控隐藏的数据或未加载出的完整评论。
- 只有小红书搜索入口不可用、搜索结果不足、或需要交叉验证时，才使用抖音、快手、马蜂窝、官方/旅游平台等来源补充。

### 美食

- 优先：大众点评/美团页面如果可访问则使用。
- 补充：携程美食、Trip.com、美食榜单、本地公众号、搜索结果。
- 注意：大众点评和美团常有验证拦截；遇到登录/扫码/验证时，优先在任务开始时的平台登录预检中一次性完成。无法核验时不要附虚假详情链接。
- 输出：店名/街区、必吃项、人均、适合时段、排队风险、来源。

### 酒店

- 优先来源：携程、飞猪、去哪儿，选择其中信息最完整、链接最稳定的平台作为主来源。
- 补充来源：酒店官网、美团/大众点评、Trip.com、同程/艺龙等只能作为补充，不能替代前三个平台。
- 输出：酒店名、位置、房型、评分、价格、是否含早、交通便利性、主来源链接、推荐理由。
- 住宿推荐表只贴一个主来源链接；用户要求比价时，再额外补查携程/飞猪/去哪儿其他平台。
- 优先使用酒店详情页；如果拿不到详情页，给平台搜索入口并标注「搜索入口，非详情页」。
- 如果主来源拿不到信息是因为登录、扫码、安全验证或风控拦截，依赖任务开始时的平台登录预检继续核验；登录后仍不可用时，换用携程/飞猪/去哪儿中的其他可用平台。
- 房价和余房必须视为动态信息。未能打开对应日期和房型页面时，标注「需用户最终确认」。
- 不要承诺「一定有房」。可以写「平台搜索线索显示可订，但需下单前复核」。

### 城际交通

- 高铁/动车：12306 为官方基准；携程、去哪儿、飞猪仅作代订价格和体验对比。
- 飞机：航司官网、携程、去哪儿、飞猪。
- 自驾：高德/百度地图估算路线、时间、高速费；油费按车型和里程估算。
- 车票、机票价格和余票强实时变化，必须标注查询时间。

### 市内交通

- 来源：高德地图、百度地图、城市地铁官网/小程序。
- 输出：打车时间和费用区间、地铁线路、步行时间。
- 地图时间按非高峰/高峰分开更好；无法确认时用区间。

## 小红书图片直通

景点和打卡点默认使用小红书搜索结果页链接：

```text
https://www.xiaohongshu.com/search_result_ai?keyword={URL编码后的关键词}&source=web_explore_feed
```

如果 AI 搜索入口不可用，降级为：

```text
http://www.xiaohongshu.com/search_result/?keyword={地名}
```

关键词优先用具体地点名，例如「紫阳街」「松兰山」「饭桶山」。不要强行附加太多后缀，避免搜索结果过窄；需要拍照机位时可另给一个「地点名 拍照」搜索入口。

美食不强行加链接；大众点评/美团不可访问时，说明原因。

## 降级策略

当任何关键平台无法访问、需要登录、被验证拦截或页面不稳定时：

1. 不编造页面内容。
2. 深度版任务开始时，先根据任务需要统一打开平台让用户扫码/登录；后续不再反复提醒登录。
3. 登录后仍无法访问时，再使用搜索结果或其他平台交叉验证，并标注「登录后仍不可访问」。
4. 标注为「搜索线索」或「需用户最终确认」。
5. 给用户提供最终复核路径，比如「下单前在 12306/携程 App 按日期复核」。

## 输出证据格式

推荐使用短标注，避免攻略变成资料库：

```markdown
来源：携程/景区官网，查询日期 2026-05-28，可信度：已核验
```

或：

```markdown
来源：WebSearch 关键词「杭州 西湖 游玩时长」，查询日期 2026-05-28，可信度：搜索线索
```
